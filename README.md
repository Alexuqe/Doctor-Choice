# Doctor Choice

Мобильное приложение для подбора врачей (педиатров): просмотр карточек, сортировка, поиск и переход в карточку врача с деталями и ценами.

---

## О проекте

**Doctor Choice** — iOS-приложение, которое помогает пользователю просматривать список педиатров, искать по имени, сортировать по цене, стажу или рейтингу и переходить в детальную карточку врача с информацией о приёме и ценах.

Данные о врачах загружаются с удалённого API, поддерживаются кэширование ответов и изображений аватаров.

---

## Стек технологий

| Категория | Технологии |
|-----------|-------------|
| **UI** | SwiftUI |
| **Архитектура** | MVVM |
| **Состояние** | Observation (`@Observable`) |
| **Язык** | Swift, async/await |
| **Сеть** | URLSession, собственный `NetworkService` |
| **Кэш** | Файловый кэш ответов API (Actor), in-memory кэш изображений (NSCache) |
| **Навигация** | NavigationStack, типобезопасный Router |
| **DI** | Собственный DIContainer, Environment |

---

## Возможности

- **Список педиатров** — карточки с аватаром, ФИО, специализацией, стажем, рейтингом и ценой.
- **Поиск** — фильтрация по имени с debounce.
- **Сортировка** — по цене, по стажу, по рейтингу (сегментированный переключатель).
- **Пагинация** — подгрузка следующих карточек при прокрутке.
- **Pull-to-refresh** — обновление списка.
- **Детальный экран врача** — переход по тапу «Записаться»: аватар, ФИО, специализация, блок цен (чат/видео/приём), кнопка записи.
- **Экран цен** — отдельный экран с детализацией цен по типам приёма (из детальной карточки).
- **Избранное** — отметка врача как избранного на карточке (состояние в модели).
- **Кэширование** — ответ API и аватары кэшируются; при повторном открытии детали аватар подставляется из кэша.

## Запуск

1. Открыть `Doctor Choice.xcodeproj` в Xcode.
2. Выбрать симулятор или устройство (iOS).
3. Запустить сборку (⌘R).

Данные загружаются с тестового JSON (URL задаётся в `Endpoint` в коде).
